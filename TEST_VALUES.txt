=== Universal Decoder 테스트 케이스 ===

## 1. URL 디코딩
### 기본
Hello%20World
→ Hello World

### 한글
%EC%9C%A0%EB%8B%88%EB%B2%84%EC%84%A4%20%EB%94%94%EC%BD%94%EB%8D%94
→ 유니버설 디코더

### 이중 인코딩 (체인 디코딩 테스트)
%2561
→ %61 → a

### 삼중 인코딩 (체인 디코딩 테스트)
%25252561
→ %2561 → %61 → a

---

## 2. HTML 엔티티
### 기본
&lt;div&gt;Hello &amp; World&lt;/div&gt;
→ <div>Hello & World</div>

### 숫자 엔티티
&#72;&#101;&#108;&#108;&#111;
→ Hello

### 16진수 엔티티
&#x48;&#x65;&#x6C;&#x6C;&#x6F;
→ Hello

## 3. Base64
### 기본
SGVsbG8gV29ybGQ=
→ Hello World

### 한글 UTF-8
7YWM7Iqk7Yq4
→ 테스트

### JSON 데이터
eyJuYW1lIjoiSm9obiIsImFnZSI6MzB9
→ {"name":"John","age":30}

---

## 4. Base64URL
### 기본 (-, _ 포함)
SGVsbG8tV29ybGRfVGVzdA
→ Hello-World_Test

### 패딩 없음
VGVzdCBCYXNlNjRVUkw
→ Test Base64URL

## 체인 디코딩 테스트 (🔗 버튼 활성화 필요)

### Case 1: Base64(URL)
SGVsbG8lMjBXb3JsZA==
→ Step 1: Base64 → Hello%20World
→ Step 2: URL → Hello World

### Case 2: Base64(Base64)
U0dWc2JHOGdWMjl5YkdRPQ==
→ Step 1: Base64 → SGVsbG8gV29ybGQ=
→ Step 2: Base64 → Hello World

### Case 3: URL(URL(URL)) - 삼중 URL 인코딩
%252548656c6c6f
→ Step 1: URL → %2548656c6c6f
→ Step 2: URL → %48656c6c6f
→ Step 3: URL → 48656c6c6f
→ Step 4: Hex → Hello

### Case 4: Base64(Hex)
NGM2NTZjNmM2Zjc1
→ Step 1: Base64 → 48656c6c6f75
→ Step 2: Hex → Hellou

## 인코딩 테스트 (Encode 모드)

### URL 인코딩
Hello World
→ Hello%20World

### HTML 인코딩
<div>Test & Example</div>
→ &lt;div&gt;Test&nbsp;&amp;&nbsp;Example&lt;/div&gt;

### Base64 인코딩
Hello World
→ SGVsbG8gV29ybGQ=

### Base64URL 인코딩
Hello-World_Test
→ SGVsbG8tV29ybGRfVGVzdA

### Hex 인코딩
Hello
→ 48656C6C6F

### CharCode 인코딩
Hello
→ [72,101,108,108,111]

### ROT13 인코딩
Hello World
→ Uryyb Jbeyq

### GZIP 인코딩
Hello World
→ Base64로 인코딩된 GZIP 압축 데이터

## 엣지 케이스 및 오류 처리

### 빈 문자열
(empty input)
→ "입력이 비어있습니다."

### 공백만

→ "입력이 비어있습니다."

### 잘못된 Base64
Invalid@Base64!
→ 자동 감지 실패 또는 디코딩 오류

### 잘못된 JWT
invalid.jwt.token
→ JWT 디코딩 실패

### 매우 긴 텍스트 (1000자 이상)
(긴 텍스트 입력)
→ 정상 처리 확인

---

## 체인 디코딩 안전장치 테스트

### 연속 동일 타입 방지
Uryyb
→ ROT13 한 번만 디코딩 후 중단
(ROT13→ROT13 방지)

### 짧은 결과 중단
SGE=
→ Base64 디코딩 → "Ha" (3자 미만으로 중단)

### 순환 패턴 감지
(A→B→A 패턴 생성하는 입력)
→ 순환 감지로 자동 중단

### JSON Pretty 제외
{"test":"value"}
→ 일반 디코딩에서는 처리하지만 체인에서는 중단
(변환이지 디코딩이 아니므로)

---

## 특수 문자 테스트

### 모든 특수문자
!@#$%^&*()_+-=[]{}|;':",.<>?/`~
→ 타입에 따라 다르게 처리

### 이모지
😀🔗🔓
→ UTF-8로 정상 처리

### 제어 문자
(탭, 개행 등)
→ 텍스트로 유지
